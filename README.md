# نظام إدارة الأعضاء - مجلس الإدارة والجمعية العمومية

تطبيق سطح مكتب (PyQt6 + SQLite) لإدارة أعضاء مجلس الإدارة والجمعية العمومية،
مع دعم:
- عضوية "Seat-Based" وحساب الأيام النشطة والتقييم تلقائيًا.
- حقول ديناميكية (EAV) قابلة للتخصيص من شاشة الإعدادات.
- رفع وتنظيم مستندات كل عضو في مجلد خاص به.
- إدارة الاجتماعات واللجان مع تشيك ليست حضور ديناميكي (يعرض فقط الأعضاء
  النشطين فعليًا في تاريخ الجلسة).
- تقارير أيام نشطة/تقييم وتقرير مالي، قابلة للتصدير إلى Excel.
- تسجيل دخول آمن (PBKDF2-HMAC-SHA256) بدون اعتمادية خارجية للتشفير.

## التشغيل

```bash
# 1) إنشاء بيئة افتراضية (اختياري لكن مستحسن)
python -m venv venv
source venv/bin/activate        # على ويندوز: venv\Scripts\activate

# 2) تثبيت المتطلبات
pip install -r requirements.txt

# 3) تشغيل التطبيق
python main.py
```

## بيانات الدخول الافتراضية
عند أول تشغيل يُنشأ مستخدم Admin تلقائيًا:
- **اسم المستخدم:** admin
- **كلمة المرور:** admin123

⚠️ يُفضّل تغيير كلمة المرور فورًا (يمكن إضافة شاشة "تغيير كلمة المرور" لاحقًا
باستخدام `AuthService.change_password()` الموجودة بالفعل في `auth/auth_service.py`).

## تحويل البرنامج إلى ملف exe واحد (بدون أي تثبيت)

المشروع جاهز للتحويل إلى ملف `.exe` واحد باستخدام **PyInstaller**، تقدر تنقله
لأي جهاز ويندوز وتشغّله مباشرة بدون Python ولا أي تثبيت.

> ⚠️ **مهم:** لازم تعمل البناء (build) من على جهاز **ويندوز فعلي** لأن PyInstaller
> بيبني الملف التنفيذي لنفس نظام التشغيل اللي بتشتغل عليه (مينفعش تبني exe
> لويندوز وانت شغال على Linux/macOS).

### الطريقة السريعة (سكريبت جاهز)
1. انسخ مجلد المشروع كامل على جهاز الويندوز.
2. شغّل ملف `build_exe.bat` بالدبل كليك.
3. بعد ما يخلص، هتلاقي الملف الجاهز هنا: `dist\MemberBoardGASystem.exe`
4. انسخ الملف ده بس (مش المجلد كله) لأي جهاز تاني وشغّله مباشرة.

### الطريقة اليدوية
```bash
pip install -r requirements.txt
pip install pyinstaller
pyinstaller app.spec --noconfirm
```
الناتج هيكون في: `dist/MemberBoardGASystem.exe`

### ملاحظات مهمة عن الـ exe
- **البيانات دائمة ومحفوظة في فولدر جنب ملف الـ exe مباشرة**: بعد أول تشغيل،
  هيتعمل تلقائيًا فولدر اسمه `AppData` في نفس مكان `MemberBoardGASystem.exe`،
  وده اللي فيه قاعدة البيانات (`app_data.db`) ومجلد المستندات (`member_documents`).
- **لو عايز تنقل البرنامج ببياناته لجهاز تاني**: انسخ ملف الـ exe **مع** فولدر
  `AppData` جنبه (مش الملف لوحده). لو نسخت الـ exe بس، هيشتغل عادي لكن
  هيبدأ بقاعدة بيانات فاضية جديدة.
- متنساش تضيف فولدر `AppData` لأي نظام نسخ احتياطي (backup) عندك، لأنه هو
  اللي فيه كل بياناتك الفعلية.
- أول تشغيل للـ exe ممكن ياخد ثواني أطول شوية من المعتاد (فك ضغط الملفات
  المؤقتة)، وده طبيعي في وضع onefile.
- لو عايز أيقونة مخصصة للبرنامج: حط ملف `app_icon.ico` في مجلد المشروع، وعدّل
  السطر الأخير في `app.spec`: `icon='app_icon.ico'`.
- لو ظهرت أي رسالة خطأ عن "missing module" عند التشغيل، ضيف اسم المكتبة الناقصة
  في قائمة `hiddenimports` بملف `app.spec` وأعد البناء.

## ملاحظات مهمة للتوسعة لاحقًا

1. **إضافة شاشة إدارة مستخدمين:** يمكن استخدام `AuthService.create_user()`
   الموجودة بالفعل لبناء شاشة "إدارة المستخدمين" (خاصة بـ Admin فقط).
2. **إدارة اللجان (UI):** الخدمة `CommitteeService` جاهزة بالكامل، لكن لا توجد
   شاشة مخصصة لها بعد — يمكن إضافتها كتبويب جديد بنفس نمط الشاشات الحالية.
3. **الحقول الديناميكية من نوع "قائمة اختيار" (choice):** حاليًا تُعرض كحقل نصي
   بسيط في `members_screen.py`؛ يمكن تحسينها لاحقًا لعرض `QComboBox` فعلي بناءً
   على `field["choices"]`.
4. **صلاحيات الأدوار (Admin/User):** الجدول والمنطق الأساسي موجودان
   (`users.role`)، لكن الشاشات الحالية لا تفرض قيودًا مختلفة بين الأدوار بعد.
5. **Phase 1 (نموذج Excel):** لم تُبنَ بعد ضمن هذا التسليم، ويمكن تنفيذها
   كخطوة تالية منفصلة (VBA + الصيغ الموضحة سابقًا في وثيقة التصميم).
